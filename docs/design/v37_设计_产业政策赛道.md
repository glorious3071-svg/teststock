# v3.7 设计:ETF 选择的"产业政策 + 估值"双驱动

> **用户洞见**:
> **"ETF 的选择除了看技术指标之外,还要结合产业发展态势和国内相关政策决定。"**
>
> 之前的 R3.v2 只考虑了"估值倒数 + 市场动能阶段",但忽略了**ETF 跟踪的指数背后是不同的行业组合**,而中国市场是**强政策市**——一个 5 年规划、一次产业扶持文件,可以让某个赛道整年跑赢大盘 30-50pp。

---

## 一、ETF 不是"技术筹码",是"产业组合"

每个 ETF 都对应一组行业暴露:

| ETF | 跟踪指数 | 主要行业暴露 | 风格 | 政策敏感性 |
|---|---|---|---|---|
| **510180** | 上证180 | 金融(40%)+ 周期(30%) + 消费(15%) | 大盘价值 | 货币/财政政策敏感 |
| **510880** | 中证红利 | 银行 + 煤炭 + 钢铁 + 公用事业 | 低估值高股息 | 利率/地产周期敏感 |
| **159901** | 深证100 | 消费(30%) + 科技(25%) + 医药(15%) + 制造 | 大盘成长 | 产业政策/消费刺激敏感 |
| **510300** | 沪深300 | 全市场覆盖 | 均衡 | 整体政策敏感 |

**关键差异**:
- **510180 vs 159901**: 一个是"老经济"代表(传统金融+周期),一个是"新经济"代表(消费+科技+医药)
- **不同的政策导向 → 不同的赛道贡献:**
  - 货币宽松/4 万亿基建 → 510180 受益(银行+建筑)
  - 家电汽车下乡/消费券 → 159901 受益(消费+制造)
  - 减税/股息率重估 → 510880 受益(高分红蓝筹)

**之前 R3 失败的根因**: 把这 3 个 ETF 当作"同质标的、只是 PE 不同",忽略了它们代表完全不同的产业逻辑。

## 三、v3.7 评分新增维度:政策赛道倾斜

### 新维度: 政策赛道得分(每月评估)

```python
def policy_track_tilt(month_state):
    """
    根据当月政策事件、即将出台的政策、宏观主线,
    给 3 个 ETF 打"赛道倾斜分":
      +2 = 强政策利好(主线)
      +1 = 中度利好
       0 = 中性
      -1 = 政策不利
      -2 = 政策强压制
    """
    tilt = {"510180": 0, "510880": 0, "159901": 0}

    # 政策标签(每月手工或抓取)
    if month_state.get("infra_stimulus"):       # 基建刺激
        tilt["510180"] += 2
        tilt["159901"] -= 1   # 资金虹吸
    if month_state.get("consumer_stimulus"):    # 消费刺激
        tilt["159901"] += 2
        tilt["510880"] += 1   # 红利白电也受益
    if month_state.get("financial_loose"):      # 金融让利/放水
        tilt["510180"] += 2
    if month_state.get("property_tighten"):     # 地产收紧
        tilt["510180"] -= 1
        tilt["510880"] -= 2   # 红利地产链承压
    if month_state.get("emerging_industry"):    # 新兴产业政策
        tilt["159901"] += 2
        tilt["510880"] -= 1
    if month_state.get("supply_side"):          # 去产能/双碳
        tilt["510880"] += 2   # 上游周期暴利
    if month_state.get("tech_selfreliance"):    # 科技自立
        tilt["159901"] += 1
    if month_state.get("monetary_tighten"):     # 紧缩周期
        tilt["510880"] += 1   # 防御
        tilt["159901"] -= 1   # 成长承压

    return tilt
```

### 综合权重公式(v3.7)

```
ETF 权重 ∝ [1/PE × 估值基础] + [动能阶段倾斜] + [政策赛道倾斜]

具体:
  raw_weight[ETF] = base_inv_pe[ETF]
                  + phase_tilt[ETF] × 0.30
                  + policy_tilt[ETF] × 0.10
  capped = clip(raw_weight, 15%, 60%)
  final = normalize(capped)
```

3 个维度都用百分比表示,最后归一化。其中:
- 估值基础(1/PE):提供静态偏好
- 动能阶段倾斜:提供时机倾斜(反弹偏成长/顶部偏防御)
- **政策赛道倾斜:提供产业方向倾斜**

## 六、待解问题

1. **政策标签如何获取?**
   - 历史回溯:可手工标注(参考国务院文件、央行季度报告、CEWC 决议)
   - 实时运行:可用 LLM 抓取财经新闻 + 政策文件
2. **标签权重 0.10 是否合理?**
   - 太低:政策维度被估值压制
   - 太高:可能过度押注单一赛道
   - 用反向验证调参
3. **是否需要扩充 ETF 池?**
   - 当前只有 510180 / 510880 / 159901 三只
   - 创业板(159915)、消费(159928)、医药(512170)、新能源 等可以纳入
   - **但用户原则:标的数 ≤ 5,且不预设核心资产**
4. **政策时滞如何处理?**
   - 政策落地 → 股价反应通常滞后 1-3 个月
   - 应该"政策出台月即倾斜",还是"政策出台后 2 月再倾斜"?
