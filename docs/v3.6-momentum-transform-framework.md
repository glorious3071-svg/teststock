# v3.6 设计：动能转换框架

## 核心原则（用户洞见）

> "决定市场行为的永远是市场本身。外部信号都需要转换成内部的动能才能决定走势。"

这意味着——同一个外部信号(过热/紧缩/宽松),在不同的市场内部状态下,会被"转换"成不同的动能(放大/吸收/反转/沉默)。

策略不能只看外部信号,要看"信号 × 当前市场动能"的乘积。

---

## 一、v3.5 失败的根因

v3.5 的逻辑是「外部信号 → 直接扣分」:

```
PE>30 → -2
新基爆款 → -1
两融激增 → -1
连涨>50% → -2
```

这是把外部信号当成因,市场行为当成果的线性逻辑。

但用户的洞见揭示:同一组外部信号,在不同市场状态下意义完全相反:

- **2007.05**: 过热信号 → 市场吸收 → 继续上涨 5 个月
- **2009.07**: 过热信号 → 市场反转 → 次月暴跌 24%

信号没变,变的是"市场把信号转换成什么动能"。

---

## 二、v3.6 双层评分架构

### Layer 1: 外部信号(基础分)—— 已在 v3.4 / v3.5 实现

- 估值(PE/PB)
- 流动性(利率/M1/信贷)
- 基本面(PMI/IVA/PPI)
- 情绪(新基/两融/媒体)
- 政策(印花税/CEWC/央行表态)

### Layer 2: 市场内部动能(转换器)—— v3.6 新增

4 个结构性条件,决定外部信号在多大程度上被"转换"成实际行情:

#### 维度 A: 政策预期已 price-in 程度

| 条件 | 转换率 | 含义 |
|------|--------|------|
| A1: 央行已加息 ≥3 次,市场已习惯紧缩 | 0.3 | 紧缩信号被吸收 |
| A2: 央行处于宽松末期,首次出现"微调/退出"措辞 | 2.0 | 紧缩信号被放大 |
| A3: 央行宽松中段,且无任何收紧迹象 | 0.5 | 紧缩信号未生效 |

#### 维度 B: 资金性质粘性

| 条件 | 转换率 | 含义 |
|------|--------|------|
| B1: 居民储蓄主导(新基连续 3 月 >1000 亿 + M1<20%) | 0.7 | 高粘性,下跌速度慢 |
| B2: 信贷溢出主导(M1>25% + 新增贷款>1 万亿) | 1.5 | 低粘性,下跌速度快 |
| B3: 杠杆资金主导(两融余额 / 流通市值 >2%) | 2.0 | 极低粘性 |

#### 维度 C: 牛市动能阶段

| 条件 | 转换率 | 含义 |
|------|--------|------|
| C1: 主升浪 <12 个月,赚钱效应仅限专业资金 | 1.5 | 散户未接盘,顶部尖锐 |
| C2: 主升浪 12-24 个月,赚钱效应扩散到中产 | 1.0 | 接盘力中等 |
| C3: 主升浪 >24 个月,赚钱效应到大妈层 | 0.5 | 散户大军接盘,顶部圆滑 |

#### 维度 D: 估值-盈利赛跑节奏

| 条件 | 转换率 | 含义 |
|------|--------|------|
| D1: PE 涨速 ≈ 盈利涨速(同步) | 0.5 | 健康 |
| D2: PE 涨速 > 盈利涨速 2 倍(领先) | 1.5 | 透支预期 |
| D3: PE 涨速 >> 盈利涨速 且盈利刚转正 | 2.0 | 严重透支 |

### 综合公式

```
过热扣分 = 外部信号基础分 × ∑(A,B,C,D 转换率)

其中 4 个维度默认值 = 1.0
满足"放大条件"时上调,满足"吸收条件"时下调
```

## 三、v3.6 评分函数(核心代码)

```python
def momentum_transformer(month_state):
    """
    输入: 月度状态(已含 v3.4/v3.5 所有字段)
    输出: 4 个维度的转换率
    """
    # A: 政策预期 price-in
    if month_state['累计加息bp'] >= 75 and month_state['持续加息月数'] >= 6:
        rate_A = 0.3  # 紧缩已 price-in
    elif month_state['央行松'] and month_state['首次出现微调']:
        rate_A = 2.0  # 宽松末期警报
    elif month_state['累计加息bp'] <= 0 and not month_state['首次出现微调']:
        rate_A = 0.5  # 宽松中段
    else:
        rate_A = 1.0

    # B: 资金粘性
    M1 = month_state['M1同比']
    新基连续 = month_state['新基连续3月>1000亿']
    信贷大 = month_state['新增贷款>1万亿']
    if 新基连续 and M1 < 20:
        rate_B = 0.7  # 储蓄长钱
    elif M1 > 25 and 信贷大:
        rate_B = 1.5  # 信贷溢出
    elif month_state.get('两融占比', 0) > 2.0:
        rate_B = 2.0  # 杠杆主导
    else:
        rate_B = 1.0

    # C: 牛市动能
    months_since_low = month_state['距上一次熊底月数']
    if months_since_low < 12:
        rate_C = 1.5  # 短反弹,顶部尖锐
    elif months_since_low <= 24:
        rate_C = 1.0
    else:
        rate_C = 0.5  # 长牛,顶部圆滑

    # D: 估值-盈利赛跑
    PE_yoy = month_state['PE同比']
    EPS_yoy = month_state.get('EPS同比', 10)  # 默认 10%
    if EPS_yoy > 0 and PE_yoy / max(EPS_yoy, 1) > 3:
        rate_D = 2.0  # 严重透支
    elif EPS_yoy > 0 and PE_yoy / max(EPS_yoy, 1) > 2:
        rate_D = 1.5
    elif abs(PE_yoy - EPS_yoy) < 5:
        rate_D = 0.5  # 同步
    else:
        rate_D = 1.0

    return rate_A, rate_B, rate_C, rate_D


def score_v36(month_state):
    # Step 1: 外部信号基础分(沿用 v3.4 逻辑)
    base_score = score_v34_external(month_state)

    # Step 2: 提取过热信号专项基础分
    overheat_base = 0
    if month_state['PE'] > 30: overheat_base -= 1
    if month_state['新基(亿)'] >= 1000: overheat_base -= 1
    if month_state['两融激增']: overheat_base -= 1
    if month_state.get('连涨5月+累计50%', False): overheat_base -= 1

    # Step 3: 动能转换
    rA, rB, rC, rD = momentum_transformer(month_state)
    multiplier = (rA + rB + rC + rD) / 4  # 平均放大因子
    overheat_actual = overheat_base * multiplier

    # Step 4: 合并(基础分中扣除过热"未放大"部分,加回放大后的部分)
    final = base_score - overheat_base + overheat_actual

    return final, {
        'base': base_score,
        'overheat_base': overheat_base,
        'multiplier': multiplier,
        'rA': rA, 'rB': rB, 'rC': rC, 'rD': rD,
        'overheat_actual': overheat_actual,
        'final': final,
    }
```

## 四、反向验证(关键时点)

| 时点 | 过热基础分 | A | B | C | D | 平均 | 实际扣分 | v3.6 决策 | 实际行情 |
|------|-----------|---|---|---|---|------|---------|-----------|---------|
| 2007.05 | -3(PE+新基+两融) | 0.3 | 0.7 | 0.5 | 0.5 | 0.50 | -1.5 | 维稳 ✅ | +60% 5个月 |
| 2007.07 | -4(+连涨) | 0.3 | 0.7 | 0.5 | 1.0 | 0.625 | -2.5 | 减一档 | +30% 3个月 |
| 2007.10 顶 | -4 | 1.0(美联储反转) | 1.5 | 1.0 | 2.0 | 1.375 | -5.5 | 强减仓+三重门 ✅ | -60% 见顶 |
| 2008.01 | -3 | 1.0 | 1.0 | 1.0 | 1.5 | 1.125 | -3.4 | 减仓 ✅ | -50% 暴跌 |
| 2009.07 | -3 | 2.0(微调首现) | 1.5(信贷) | 1.5(<12月) | 2.0(透支) | 1.75 | -5.3 | 强减仓 ✅ | -24% 暴跌 |

**关键:**

- **2007.05**: 4 个维度全部"吸收"→ 0.5 倍 → 过热信号几乎不扣分 → 维稳
- **2009.07**: 4 个维度全部"放大"→ 1.75 倍 → 过热信号 3.5 倍杀伤 → 强减仓
- **2007.10 顶部**: 美联储反转 + 透支严重 → 配合卖出三重门 → 见顶减仓
- **2008.01**: 政策已 price-in 但估值透支 → 中等扣分 → 减仓

## 七、本质总结

| 版本 | 逻辑 | 特点 |
|------|------|------|
| v3.4 | 外部信号 → 直接定价 | 线性,顶部失灵 |
| v3.5 | 外部信号 + 领先指标 → 直接定价 | 更多线性变量,过度敏感 |
| v3.6 | 外部信号 × 市场动能转换 → 定价 | 乘积,贴近现实 |

这是从"看温度计"到"看体温背后的免疫系统"的进化。
