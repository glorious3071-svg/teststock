# V5.0 评分卡 — 累计加息 / 累计加准 计算说明

> 对应字段：`rate_cum_bp_12m`、`rrr_cum_pp_12m`
> 代码位置：`backtest/scorecard_adapter.py` + `backtest/scorecard.py`
> 维度：**流动性 [-4, +4]**
> 数据口径：as_of_date = 评分年份 year-1 的 12-31，严防上帝视角

---

## 一、累计加息 `rate_cum_bp_12m`（单位 bp）

### 计算公式

```
rate_cum_bp_12m = (本期 SHIBOR_3M − 一年前 SHIBOR_3M) × 100
```

> SHIBOR 单位是 `%`，差值乘以 100 转换成 bp（基点）。
> 例：3M 利率从 3.20% → 2.50%，差 -0.70 pp = **-70 bp**

### 实现源码（`_shibor_3m_cum_bp`，line 359）

1. 取 `shibor_daily` 中 `trade_date ≤ snapshot_date` 的最新一条 `rate_3m`（当前点）
2. 取 `trade_date ≤ (snapshot_date 一年前)` 的最新一条 `rate_3m`（一年前点）
3. 两者直接相减 × 100

> **不是滚动累计加息次数，而是"末点 vs 起点"的净变化**，效果上等价于过去 12 个月的累计净加/降息幅度。

### 数据回落顺序

```
shibor_daily.rate_3m   ← 主源（2006-10-08 起）
  ↓ 若 None
chibor_daily.rate_3m   ← fallback（2004-05-24 起）  ← 新增
  ↓ 若 None
annual_snapshot.shibor_3m_yoy_bp  ← 终极兜底
```

### 触发规则（`scorecard.py:136-140`）

| 条件 | 方向 | 分值 |
|---|---|---|
| `rate_cum_bp_12m > 150` | 风险 | **+1**（叠加） |
| `rate_cum_bp_12m > 100` | 风险 | **+1**（≥150 时累计 +2） |
| `rate_cum_bp_12m < −100` | 机会 | **−2** |

> 注意：>150 同时满足 >100，所以会拿到 +2。<-100 只触发一次 -2。

### 2006 评分卡实例

- snapshot_date = 2005-12-31
- 当期：CHIBOR 3M = **2.50%** (2005-12-31)
- 一年前：CHIBOR 3M = **3.20%** (2004-12-31)
- 累计：`(2.50 − 3.20) × 100 = −70 bp`
- 触发：未触发 −100bp 阈值 → **0 分**

## 二、累计加准 `rrr_cum_pp_12m`（单位 pp，百分点）

### 计算公式

```
rrr_cum_pp_12m = ∑(rrr_change_pp) over (snapshot_date−1y, snapshot_date]
                 where inst_type IN ('large', 'all')
```

> **直接对 12 个月内的 RRR 调整事件求代数和**，正为加准，负为降准。

### 实现源码（`_rrr_cum_pp_12m`，line 262）

```sql
SELECT COALESCE(SUM(rrr_change_pp), 0)
FROM cn_rrr_changes
WHERE effective_date > snapshot_date - 1y
  AND effective_date <= snapshot_date
  AND (inst_type='large' OR inst_type='all')
```

- `cn_rrr_changes` 是事件表，每次央行宣布调整存款准备金率写入一行
- `rrr_change_pp` 是单次变动幅度（如 +0.50 表示加准 50bp，−1.00 表示降准 100bp）
- 只统计 `large`（大型机构）或 `all`（一刀切）口径，剔除小行差别化调整避免重复
- 没有事件时返回 **0.0**（不是 None）

### 触发规则（`scorecard.py:141-144`）

| 条件 | 方向 | 分值 |
|---|---|---|
| `rrr_cum_pp_12m > 3` | 风险 | **+1** |
| `rrr_cum_pp_12m < −1` | 机会 | **−1** |

### 2006 评分卡实例

- 窗口：2005-01-01 ~ 2005-12-31
- 2005 年 RRR 调整事件（large/all）：**0 次**
- 累计：`SUM = 0.00 pp`
- 触发：未触发 → **0 分**

## 三、两个口径的设计差异

| 维度 | 累计加息 `rate_cum_bp_12m` | 累计加准 `rrr_cum_pp_12m` |
|---|---|---|
| 数据形态 | **连续**（每日 SHIBOR） | **事件**（央行公告） |
| 计算方式 | 末点 − 起点 | 区间内事件求和 |
| 单位 | bp（×100 转换） | pp（直接累加） |
| 缺数行为 | 返回 None，转用 yoy_bp 兜底 | 默认 0.0（无事件即 0） |
| 反映对象 | 银行间市场实际利率水平 | 央行政策操作意图 |

> **为什么不用一年存款基准利率算累计加息？**
> 1Y 定存基准是离散事件（一年调整 1-3 次），用末点−起点会丢失中间路径信息；SHIBOR 反映真实流动性松紧，连续日频，更适合做累计 bp 计算。`deposit_1y_rate` 在评分卡里单独看绝对水平（>3.5% / <2.5%），不算累计变动。

## 五、可改进方向（备忘）

1. **当前用 SHIBOR/CHIBOR 3M 代理政策利率**，更严谨可加入 OMO 7D 逆回购利率作为锚（2013 后才可用）
2. **存款基准利率累计变动**当前未独立入分项，可考虑增设 `deposit_1y_cum_bp_12m` 区分基准利率（受央行直接管控）与市场利率
3. **RRR 加权**：目前 large/all 等权，可按银行体系存款规模加权

## 六、`deposit_1y_rate` 两段拼接口径（2026-06 新增）

### 背景

央行 1Y 定存基准利率自 **2015-10-24** 最后一次降息至 **1.50%** 后冻结，不再公布调整。直接读 `cn_deposit_rate` 会导致 2015 后所有 snapshot 永远返回 1.50%，触发评分卡 `<2.5%` 机会-1 阈值——**信号失真**。

### 调整方案（已在 `scripts/check_liquidity_features.py` 实现）

```
deposit_1y_rate(as_of):
    if as_of ≤ 2015-10-24:
        return latest cn_deposit_rate.rate_after_pct
    else:
        return AVG(shibor_daily.rate_1y) over [as_of-30d, as_of]
```

`cn_deposit_rate` 原表完全保留（事件原始记录），仅在读取层做两段拼接。

### 验证（11 个 benchmark 年份）

| snap | deposit_1y | 来源 | 触发评分阈值 |
|---|---|---|---|
| 2005-12-31 | 2.25% | cn_deposit_rate@2004-10-29 | 无 |
| 2007-12-31 | 4.14% | cn_deposit_rate@2007-12-21 | **>3.5% 风险+1** |
| 2009-12-31 | 2.25% | cn_deposit_rate@2008-12-23 | **<2.5% 机会-1** |
| 2011-12-31 | 3.50% | cn_deposit_rate@2011-07-07 | 临界 |
| 2014-12-31 | 2.75% | cn_deposit_rate@2014-11-22 | 无 |
| 2015-12-31 | **3.35%** | SHIBOR 1Y 30d | 无 |
| 2017-12-31 | **4.70%** | SHIBOR 1Y 30d | **>3.5% 风险+1**（金融去杠杆顶部，方向正确）|
| 2019-12-31 | **3.12%** | SHIBOR 1Y 30d | 无 |
| 2021-12-31 | **2.74%** | SHIBOR 1Y 30d | 无 |
| 2023-12-31 | **2.62%** | SHIBOR 1Y 30d | 无 |
| 2025-12-31 | **1.65%** | SHIBOR 1Y 30d | **<2.5% 机会-1**（货币宽松底部，方向正确）|

### 已知偏差

1. **SHIBOR 1Y 系统性高于零售存款（重叠期 2006-10 ~ 2015-10 平均 +1.03 pp）**，但利差不稳定：std=0.78 pp，区间 -0.40 ~ +2.28 pp，随利率周期变化（紧缩期利差大，宽松期利差小甚至倒挂）。
2. **2015-10-24 口径切换处出现 +1.91 pp 跳变**（央行末值 1.50% → SHIBOR 30d 均值 3.41%）。横截面年度评分判分方向不受影响；做时序分析需平移校准。
3. **2015 之后硬阈值方向仍正确**：11 个 benchmark 年评分方向 100% 命中（2007/2017 触发紧缩，2009/2025 触发宽松，与真实政策周期吻合）。

### 长期数据评估（2026-06）

调研「能否用更长历史数据优化两段拼接逻辑」，结论：**保持现行方案，不做优化**。

| 候选方案 | 数据来源 | 评估结论 |
|---|---|---|
| 用 1Y 贷款基准延伸至 1991-2019 | `akshare.macro_bank_china_interest_rate`（218 行）| 维度不匹配（贷款 vs 定存），且 2002 后 V5.0 不需要 1991-2001 历史 |
| 固定利差校准 SHIBOR 代理 | 重叠期 mean 利差 1.03 pp 平移 | 利差 std=0.78 pp 时变剧烈，校准后 2025-12 ≈ 0.62% 明显失真 |
| SHIBOR 1Y 滚动 5Y P25/P75 动态阈值 | shibor_daily 完整 4871 条 | 2015-12 切换点滚动窗口仍是定存数据（P25=4.40% / P75=5.00%），SHIBOR 3.41% 反被判机会，加重跨段误判 |
| 引入 NCD 1Y 同业存单收益率 | akshare 无直接接口（需 Wind/iFinD） | 数据源不可得，后续可选 |

**核心约束**：央行自 2015-10-24 停止公布 1Y 定存基准；同业市场（SHIBOR/CHIBOR）与零售定存的利差时变不可固定校准。**当前两段拼接 + 硬阈值在所有 benchmark 年方向 100% 正确，性价比不支持引入复杂度**。

### 涉及数据表

- `cn_deposit_rate`：种子 CSV `data/cn_deposit_rate.csv`，26 行，覆盖 2002-2015
- `shibor_daily.rate_1y`：Tushare 自动同步，2006-10 至今
